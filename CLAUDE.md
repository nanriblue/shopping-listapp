# CLAUDE.md

## Git 커밋 규칙
- 작업 완료 후 항상 커밋하고 GitHub에 푸시할 것
- 커밋 메시지는 한글로 작성하고, 변경 내용을 명확하게 설명할 것

## 프로젝트 구조
- 프로젝트 폴더 구조는 항상 깔끔하게 정리할 것 (src, public 등 역할별로 분리)

## 문서 관리
- README.md를 항상 최신 상태로 유지할 것 (프로젝트 설명, 실행 방법 포함)

## Git 관리
- .gitignore 파일을 관리하여 node_modules, .env 등 불필요한 파일은 GitHub에 올리지 않을 것

## 원격 저장소
